- 검색창 + 정렬/필터 접기 버튼 row 구성 완료.

### 3. 전체게임보기 검색
- 검색창 입력 시 전체게임 리스트 필터링 작동.
- 초성/한글 스마트 검색 기반 일부 적용.
- URL query `?search=` 기반 검색어 주입 구조 있음.

### 4. 전체게임보기 정렬
- 이름 / 난이도 / 평점 정렬 select 존재.
- 다중 정렬 작동.
- 마지막으로 건드린 정렬 기준이 1순위로 올라가는 구조 구현.
- `activeSortKeys` 기반으로 정렬 우선순위 관리.
- 비활성 `-` 기준은 우선순위에서 빠지고 뒤로 이동.
- 화면상 select 순서도 활성 우선순위대로 재배치됨.
- 현재 문제: select 안 option 텍스트에 `1.난이도` 같은 번호가 들어가면, 드롭다운 열린 상태에서도 현재 선택 항목에 번호가 같이 보임. 네이티브 `<select>` 특성상 닫힌 표시와 열린 목록의 같은 option 텍스트를 완전히 다르게 분리하기 어려움.

### 5. 전체게임보기 필터
- 체감 난이도 필터 작동.
- 게임 유형/메커니즘 필터 작동.
- 체감 난이도 select는 placeholder + 전체 옵션 구조로 바뀜.
- 전체 선택 시 다시 placeholder 표시로 복귀하는 로직 적용.
- 게임 유형도 같은 방식 적용.
- 필터 label은 `🔎 필터`, select는 `🎯 체감 난이도`, `🎲 게임 유형` 톤으로 맞추는 중.

### 6. 전체게임보기 컨트롤 영역 polish
- 정렬/필터는 같은 카드 안에서 2줄 구조:
  - `↕ 정렬 [정렬 select들]`
  - `🔎 필터 [체감 난이도] [게임 유형]`
- 검색 row는 아래 독립:
  - `[검색창] [▲/▼ 정렬·필터 버튼]`
- CSS override가 많이 누적되어 있어, 다음 방에서 정리 필요.

## 현재 남은 문제 / 다음 방 첫 작업

### 최우선: 정렬 우선순위 번호 표시 방식 재설계

현재 방식:
```text
1.난이도 ↑
2.이름 ↑
평점 -
```

문제:
- 번호가 option 텍스트 안에 들어가 있음.
- 그래서 select를 열었을 때도 현재 선택 항목이 `1.난이도 ↑`로 보임.
- 사용자가 원하는 느낌은 닫힌 상태에서는 우선순위가 보이되, 선택 목록은 그냥 `난이도 - / 난이도 ↑ / 난이도 ↓`처럼 깔끔해야 함.

추천 다음 구조:
```text
[1] [난이도 ↑]
[2] [이름 ↑]
    [평점 -]
```

즉 **우선순위 번호를 select 텍스트 안에 넣지 말고, select 앞의 작은 badge/span으로 분리**하는 방식.

다음 방에서 바로 할 일:
1. `owned-games.html` 정렬 select 주변 구조 변경
2. `updateSortOptionLabels()`에서 option 텍스트는 번호 없이 `이름 ↑`, `난이도 ↓`만 유지
3. 우선순위 번호는 별도 badge로 JS에서 갱신
4. CSS는 정렬 badge + select pill이 한 덩어리처럼 보이게 조정

## 중요한 작업 방식

- UI/CSS는 바로 디테일 패치하지 말고 먼저 구조 시안 A/B/C로 판단.
- 구조 확정 후 HTML/CSS/JS 수정 범위를 한 번에 묶어 진행.
- 마지막에만 여백/높이/선/그림자 polish.
- 단순 몇 줄 수정은 부분패치로 안내.
- 전체 구조가 꼬였거나 override가 많아지면 파일 교체 방식 고려.
- 긴 코드 전문은 채팅에 붙이지 말고 파일 생성/교체 방식 선호.

## 감안할 점
