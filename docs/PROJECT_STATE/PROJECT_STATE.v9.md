# PROJECT_STATE.v9

## 현재 상태 요약

코티지보드 홈페이지 작업은 `owned-games.html`의 전체게임보기 페이지 UI 조정 단계다.

현재 핵심 방향은 다음과 같다.

- 전체게임보기 페이지는 추천게임 UI가 아니라 DB 탐색 화면이다.
- 검색은 가장 자주 쓰는 기능이므로 항상 고정 노출한다.
- 정렬/필터는 보조 탐색 기능이므로 기본 접힘 구조로 둔다.
- 정렬/필터를 건드릴 때는 CSS 미세수정부터 반복하지 않고, 먼저 레이아웃 구조 시안을 비교한 뒤 확정된 구조만 묶어서 수정한다.

## 이번 방에서 확정된 UI workflow

앞으로 코티지보드 UI 작업은 다음 순서로 진행한다.

1. 구조 시안 빠른 비교
   - A안 / B안 / C안 형태로 레이아웃과 UX 구조를 먼저 판단한다.
   - 코드 수정 전에 구조 자체가 맞는지 먼저 본다.

2. 방향 확정
   - 사용자가 선택한 구조를 기준으로만 구현한다.
   - 기존 구조를 억지로 유지하면서 CSS를 덕지덕지 수정하지 않는다.

3. HTML/CSS/JS 묶음 수정
   - 구조가 확정된 뒤 필요한 파일 수정 범위를 한 번에 정리한다.
   - 단순 몇 줄 수정은 부분수정으로 안내한다.
   - 파일 전체가 꼬였거나 구조가 크게 바뀌는 경우에만 전체 파일 교체/생성 방식을 쓴다.

4. 마지막 polish
   - 여백, 높이, 선, 그림자, 색감 등 디테일은 마지막 단계에서만 조정한다.

## owned-games.html 현재 목표 구조

전체게임보기 페이지의 상단 탐색 구조는 다음 방향으로 확정한다.

닫힘 기본 상태:

```text
[게임 검색창 고정]

▼ 정렬·필터
```

열림 상태:

```text
[게임 검색창 고정]

▲ 정렬·필터

↕ 정렬 [이름] [난이도] [평점]

🔎 필터 [체감 난이도] [메커니즘]
```

## 다음 작업 우선순위

1. `owned-games.html` 구조 수정
   - 검색창은 `owned-toolbar` 밖에 둔다.
   - `owned-toolbar`에는 정렬과 필터만 넣는다.
   - `owned-toolbar`는 기본 접힘 상태로 시작한다.
   - `ownedToolsToggle` 버튼을 검색창 아래, toolbar 위에 둔다.

2. `script.js` 수정
   - `ownedToolsToggle` 클릭 시 `ownedToolbar.is-collapsed` 토글.
   - 버튼 텍스트를 `▼ 정렬·필터` / `▲ 정렬·필터`로 변경.
   - 체감 난이도는 현재 방향상 compact select 방식 유지 가능.
   - 메커니즘도 select 유지.

3. `style.css` 수정
   - 검색창은 고정 검색바처럼 명확하게 보이게 한다.
   - 정렬·필터 toggle은 얇은 한 줄 버튼으로 만든다.
   - 접힌 toolbar는 `max-height:0`, `opacity:0`, `pointer-events:none`.
   - 열린 toolbar는 정렬/필터만 보여준다.
   - 필터/정렬 카드의 높이는 최대한 압축한다.

4. 전체게임 카드 간격 최종 polish
   - 카드 gap, padding, 이미지 크기, 메타 pill 높이만 마지막에 조정한다.

5. 작업 완료 후
   - `PROJECT_STATE.v10`
   - `PROJECT_STRUCTURE.v7`
   로 갱신한다.

## 주의사항

- 기능 추가 금지.
- 이번 단계는 UI 구조 정리와 CSS polish 중심.
- 검색 기능 자체는 건드리지 않는다.
- 정렬/필터 로직도 기존 기능 유지가 우선이다.
- 불필요한 CSS 덮어쓰기 누적을 줄인다.
- 새 구조가 확정되기 전에는 최종처럼 디테일 패치를 반복하지 않는다.
